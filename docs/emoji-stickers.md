# カスタム絵文字・スタンプ運用ガイド (Phase 7 ⑤)

SelfMatrix クライアント (Cinny fork) のカスタム絵文字/スタンプは、fork 元の Cinny が実装済みの image pack 機能 (MSC2545。2026-07 に Matrix 正式仕様へマージ) をそのまま使います。**SelfMatrix 固有の実装はありません** — 本ドキュメントは「どう運用するか」の型と操作手順だけを定めます。

## 1. 仕組みの要点

パックには 3 つの置き場所があります。

| 種類 | 置き場所 | 見える範囲 |
| --- | --- | --- |
| 個人パック | 自分のアカウント (個人設定) | 自分だけ・全ルームで使える |
| ルームパック | チャンネル (ルーム設定) | そのチャンネルの参加者 |
| お気に入り登録 | 個人設定 | 参加中ルームのパックを自分の全ルームで使えるようにする |

**推奨運用 (Discord の「サーバー絵文字」相当)**: サーバーに「絵文字倉庫」のようなチャンネルを 1 つ作り、そこにルームパックとしてサーバー共通の絵文字/スタンプを置きます。メンバーは各自そのパックを「お気に入り」登録すれば、どのチャンネルでも使えるようになります。

## 2. サーバー共通パックを作る (運用者/モデレーター向け)

1. 絵文字倉庫にするチャンネルを開き、ヘッダーのメニューから「チャンネル設定」→「絵文字とスタンプ」タブ
2. 「新しいパック」に名前を入れて「作成」
3. パックの「表示」を押して編集画面へ。**この編集画面は現時点でほぼすべて英語表示です** (以下のボタン名は代表例。`Pack`/`Name`/`Save`/`Cancel`/`Shortcode:` など編集画面内の文言全般が対象 — fork の i18n 残課題):
   - `Upload Images` の `Select` で画像を選ぶ (複数可)。ショートコードはファイル名から自動で付き、空白はハイフンに、重複には連番が付きます
   - 各画像の `Edit` でショートコードや説明を調整。パック単位/画像単位で `Images Usage` (絵文字/スタンプ/両方) を切替
   - 変更は下部バナーの `Apply Changes` を押すまで確定されません
4. 画像の目安: 形式は一般的な画像 (png/gif/webp 等)。クライアント側にサイズ上限のハードコードはなく、サーバーのアップロード上限に従います — とはいえ**絵文字は 256px 四方・数百 KB 以内を推奨** (メッセージ中に小さく描画されるため大きい画像は無駄に帯域を食います)

## 3. メンバーが使う

- **絵文字**: メッセージ入力欄の顔アイコンで絵文字ボードを開くか、本文に `:` に続けてショートコードを打つと補完が出ます (Tab で先頭候補を確定)
- **スタンプ**: 入力欄のスタンプアイコン (画面幅が狭いときは顔アイコンから「スタンプ」タブ)。**クリックした瞬間に送信されます** (入力欄には入りません)
- **サーバー共通パックを全チャンネルで使う**: 個人設定 (サイドバーの自分のアバター) →「絵文字とスタンプ」→「お気に入りのパック」の「選択」→ 絵文字倉庫チャンネルのパックにチェック → 「保存」→「変更を適用」
- **自分専用の絵文字**: 同じ画面の「デフォルトパック」の「表示」から、§2 と同じ編集画面で追加できます

## 4. 権限とモデレーション

- ルームパックの編集に必要な power level は、既定でルームの `state_default` (= 50、モデレーター以上) です。緩めたい/締めたい場合は「チャンネル設定」→「権限」→「その他」の**「絵文字とスタンプの管理」**で変更できます
- **「スタンプの送信」は別の権限項目**です — こちらは権限タブの「メッセージ」グループにあり、パック管理 (「その他」グループ) とは独立に制御できます
- 不適切な画像への対応: パックの編集画面で該当画像を削除して `Apply Changes`、またはルーム設定からパックごと削除します。すでに送信されたメッセージ内の画像は通常のメッセージ削除で対応し、必要なら運用 runbook (docs/operations.md) の手順でユーザー対応まで行ってください
- メディアは Matrix のメディアリポジトリに保存され、federation 先にキャッシュされ得ます。「消せば完全に消える」とは考えないでください

## 5. 既知の注意

- **パック編集画面 (名前・画像アップロード・ショートコード編集) は未翻訳で英語表示です**。タブや一覧までは日本語化済み。編集画面の i18n 化は fork の残課題として管理しています
- MSC2545 は 2026-07 に正式仕様になったばかりで、他クライアントの対応はまちまちです。本プロジェクトは「参加者全員が SelfMatrix クライアントを使う」前提 (requirements.md §7) なので実用上の問題はありません
